# Дизайн: панель кнопок-сценариев в пустом состоянии чата

Дата: 2026-07-05
Статус: утверждён

## Цель

В пустом состоянии чата (блок подсказки «Спросите что-нибудь об этой странице…»
в `extension/src/ChatPanel.tsx`) показать панель из трёх кнопок, демонстрирующих
новые инструменты агента. Клик отправляет готовый вопрос в чат.

## Кнопки

| Надпись | Отправляемый текст |
|---|---|
| 📋 Скопируй страницу в Markdown | `Скопируй страницу в markdown` |
| 🔊 Озвучь страницу | `Кратко перескажи страницу и озвучь пересказ` |
| 🔗 Открой первую ссылку и перескажи | `Открой первую ссылку на странице и перескажи её содержимое` |

Для озвучки в пустом чате ответа ещё нет, поэтому кнопка просит пересказ
страницы с озвучкой (решение согласовано с пользователем).

## Реализация

Только `extension/src/ChatPanel.tsx`:

- Константа `DEMO_PROMPTS: { label: string; prompt: string }[]` рядом с
  существующей `QUICK_PROMPTS`.
- В блоке пустого состояния (`messages.length === 0`), только при
  `pageReadable`, под текстом подсказки — вертикальная колонка кнопок
  (`flex flex-col gap-2`), каждая на всю ширину, стиль как у пилюль
  `QUICK_PROMPTS` (рамка, `bg-secondary`, hover `bg-accent`), выравнивание
  текста по левому краю.
- Клик — существующий `onSend(prompt)`. После отправки появляется сообщение,
  пустое состояние исчезает само; отдельная логика скрытия не нужна.
- Существующий ряд `QUICK_PROMPTS` над полем ввода не меняется.
- На нечитаемых страницах (`pageReadable === false`) панель не показывается —
  как и текущая подсказка с примерами.

## Вне скоупа

- Изменения бэкенда, новых компонентов-файлов, иконок-библиотек (эмодзи в
  строке надписи).

## Проверка

`cd extension && npx tsc -b && npm run build` без ошибок; ручная проверка в
Chrome: кнопки видны в новом чате, клик отправляет вопрос, панель исчезает.
