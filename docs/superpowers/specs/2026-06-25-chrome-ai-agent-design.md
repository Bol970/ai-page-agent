# Дизайн: AI-агент как расширение Chrome (анализ страницы + поиск EXA)

Дата: 2026-06-25
Статус: утверждён

## Цель

Учебное задание курса «Разработка с AI на практике для всех»:

1. AI-агент в виде расширения Chrome, который получает содержимое открытой
   веб-страницы и отвечает на вопросы про неё.
2. Добавить агенту инструмент — поиск в интернете через API сервиса **EXA**.

Агент строится на **LangGraph**.

## Архитектура

```
┌─────────────────────────────────────┐
│  Chrome Extension (Manifest V3)      │
│  React + TS + Vite + Tailwind +      │
│  shadcn/ui                           │
│                                      │
│  content script ──► innerText стр.   │
│  popup (чат UI)  ──► вопрос + текст  │
└───────────────┬──────────────────────┘
                │ HTTP POST /chat (JSON)
                ▼
┌─────────────────────────────────────┐
│  Python backend  (FastAPI :8000)     │
│  LangGraph агент (ReAct)             │
│   • LLM: OpenRouter (base_url+key)   │
│   • инструмент: exa_search           │
│   • контекст: текст страницы         │
│   • память диалога: checkpointer     │
└─────────────────────────────────────┘
```

Расширение читает страницу и рисует чат. Вся логика агента и API-ключи живут в
Python-сервере; браузер ключей не видит.

**Содержимое страницы — это контекст, а не инструмент.** Текст страницы физически
доступен только в браузере, сервер сам его достать не может. Поэтому расширение
присылает текст вместе с вопросом, и агент видит его в системном контексте.
**EXA-поиск — настоящий tool**, который агент вызывает сам, когда на странице нет
ответа или нужна свежая информация из интернета. Это ровно то, что требует задание.

## Компоненты

### Расширение (`extension/`) — React + TypeScript + Vite + Tailwind + shadcn/ui

```
extension/
  src/
    popup/
      Popup.tsx          ← чат-интерфейс на shadcn/ui
      main.tsx
    components/ui/        ← скопированные shadcn-компоненты
    content.ts           ← извлекает innerText / title / url страницы
    lib/api.ts           ← fetch на localhost:8000/chat
  index.html             ← точка входа попапа
  manifest.json          ← MV3 (permissions: activeTab, scripting; host localhost:8000)
  vite.config.ts         ← сборка через @crxjs/vite-plugin
  tailwind.config.js
  components.json         ← конфиг shadcn/ui
  package.json
```

Компоненты shadcn для чата: `Button`, `Textarea`, `ScrollArea`, `Card`
(пузыри сообщений), опционально `Separator`/`Avatar`. Тёмная тема.
Сборка: `npm run build` → папка `dist/`, грузится в Chrome как распакованное
расширение.

### Бэкенд (`backend/`) — Python + FastAPI + LangGraph

```
backend/
  main.py            ← FastAPI: POST /chat, CORS для расширения
  agent.py           ← сборка LangGraph-агента (create_react_agent)
  tools.py           ← инструмент exa_search (exa-py)
  .env               ← OPENROUTER_API_KEY, OPENROUTER_MODEL, EXA_API_KEY
  .env.example
  requirements.txt
```

LLM подключается как OpenAI-совместимый клиент:
`ChatOpenAI(base_url="https://openrouter.ai/api/v1", api_key=OPENROUTER_API_KEY,
model=OPENROUTER_MODEL)`.

## Поток данных и память

1. Пользователь открывает popup, пишет вопрос.
2. `popup` запрашивает у content-скрипта текст активной вкладки → POST `/chat`:
   `{ thread_id, question, page: { title, url, text } }`.
3. Сервер кладёт текст страницы в системный промпт, вопрос — в сообщения.
4. Агент решает: ответить по странице или вызвать `exa_search`.
5. История диалога хранится в LangGraph **checkpointer (MemorySaver)** по
   `thread_id` (один thread = одна вкладка) — можно задавать уточняющие вопросы
   подряд.
6. Ответ возвращается в popup и добавляется в ленту сообщений.

Текст страницы обрезается до ~12000 символов, чтобы влезть в контекст и не
переплачивать (для учебного демо достаточно).

## Инструмент EXA

`exa_search(query: str)` — вызывает EXA `search_and_contents`, возвращает 3–5
результатов (заголовок, url, краткий текст). Агент сам формулирует запрос и
вызывает инструмент, когда на странице нет ответа или нужна свежая информация.
В ответе агент ссылается на источники.

## Обработка ошибок

- Нет обязательных ключей (`OPENROUTER_API_KEY`, `EXA_API_KEY`) → сервер на старте
  пишет понятную ошибку и не запускается / возвращает 500 с пояснением.
- Сбой EXA или LLM → агент/эндпоинт возвращает текстовое сообщение об ошибке,
  popup показывает его в ленте.
- Недоступен бэкенд → popup показывает «сервер не запущен».

## Тестирование

- `pytest`: юнит-тест инструмента `exa_search` (мок сети) и тест сборки агента.
- Ручной чек: открыть любую статью → «о чём эта страница?» (ответ из контекста);
  затем «найди свежие новости по теме» (агент вызывает `exa_search`).

## Принятые решения (вне кода)

- Архитектура: Python-бэкенд + расширение (ключи скрыты от браузера).
- LLM: OpenRouter (OpenAI-совместимый), ключ и модель — в `.env`.
- Объём: рабочее демо + история диалога.
- Фронтенд: shadcn/ui (React + TS + Vite + Tailwind), сборка через
  `@crxjs/vite-plugin`.
