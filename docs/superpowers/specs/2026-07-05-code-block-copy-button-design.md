# Дизайн: кнопка «Копировать» на код-блоках ответов

Дата: 2026-07-05
Статус: утверждён

## Цель

Ответы ассистента рендерятся как HTML (marked + DOMPurify), код-блоки —
`<pre><code>…</code></pre>`. Инструмент `page_to_markdown` выводит Markdown
страницы в такой блок, но скопировать его можно только вручную. Добавить на
каждый блочный код-фрагмент кнопку «Копировать», пишущую его текст в буфер
обмена.

## Реализация

Только `extension/src/ChatPanel.tsx`.

Новый компонент `AssistantHtml({ content }: { content: string })`:

- Рендерит `<div ref={ref} className="assistant-html min-w-0 flex-1 pt-0.5"
  dangerouslySetInnerHTML={{ __html: renderMarkdown(content) }} />`
  (заменяет текущий инлайновый `<div className="assistant-html …">` в ветке
  не-error сообщения ассистента).
- `useEffect` с зависимостью `[content]`:
  1. Удаляет ранее добавленные кнопки (querySelectorAll по data-атрибуту) —
     защита от дублей при ре-рендере.
  2. Для каждого `pre` в контейнере: ставит `pre.style.position = "relative"`,
     создаёт `<button>` с классами (литеральные строки, чтобы Tailwind их
     увидел) в правом верхнем углу, текст «Копировать», помечает
     `data-copy-btn`.
  3. Обработчик клика: `navigator.clipboard.writeText(code)`, где `code` —
     `innerText` вложенного `<code>` (или самого `pre`, если `code` нет).
     На успех — текст кнопки «Скопировано ✓» на 1500 мс, затем обратно.
     На ошибку (нет Clipboard API / отказ) — «Не удалось» на 1500 мс.
  4. Возвращает cleanup, снимающий добавленные кнопки.

Таймер сбрасываемого текста хранить в замыкании обработчика; при повторном
клике до истечения — сбрасывать предыдущий таймер (чтобы не мигало).

Место вставки: в `messages.map(...)`, ветка ассистента `!m.isError` —
внешний `<div className="min-w-0 flex-1">` остаётся, внутренний
`assistant-html`-div заменяется на `<AssistantHtml content={m.content} />`,
аудиоплеер (`extractAudioUrls`) — без изменений под ним.

## Обработка ошибок

- Нет `navigator.clipboard` или `writeText` отклонён → надпись «Не удалось»,
  ничего не роняем.
- Ветка error-сообщений (`m.isError`) кнопок не получает (там нет HTML-рендера).

## Вне скоупа

- Копирование всего сообщения одной кнопкой.
- Кнопки на inline-`code` (только блочные `<pre>`).
- Подсветка синтаксиса, изменения `markdown.ts`, разрешения в манифесте.

## Проверка

`cd extension && npx tsc -b && npm run build` без ошибок; ручная проверка в
Chrome: «Скопируй страницу в markdown» → на код-блоке видна кнопка, клик
кладёт Markdown в буфер, надпись меняется на «Скопировано ✓».
