# Дизайн: история чатов по страницам (закрепление + теги)

Дата: 2026-06-25
Статус: утверждён
Базируется на: [2026-06-25-chrome-ai-agent-design.md](2026-06-25-chrome-ai-agent-design.md),
[2026-06-25-sidepanel-chatgpt-html-design.md](2026-06-25-sidepanel-chatgpt-html-design.md)

## Цель

Добавить постоянную историю чатов:
1. Чаты сохраняются и группируются **по странице** (нормализованный URL).
2. Несколько чатов на страницу + создание новых.
3. **Закрепление** чатов (pinned — наверху списка).
4. **Теги** на чатах + фильтр списка по тегу.

Хранилище — **SQLite на бэкенде**; БД — единственный источник правды.

## 1. Хранилище (SQLite)

Файл `backend/chats.db`. Подключение через стандартный `sqlite3`.

**Таблица `chats`:**
- `id` TEXT PK (uuid4)
- `page_url` TEXT — нормализованный URL (без query и hash)
- `page_title` TEXT
- `title` TEXT — заголовок чата (из первого вопроса, обрезка ~60 симв.)
- `pinned` INTEGER (0/1, по умолчанию 0)
- `tags` TEXT — JSON-массив строк (по умолчанию `[]`)
- `created_at` TEXT (ISO), `updated_at` TEXT (ISO)

**Таблица `messages`:**
- `id` TEXT PK (uuid4)
- `chat_id` TEXT FK → chats.id (ON DELETE CASCADE)
- `role` TEXT (`user` | `assistant`)
- `content` TEXT
- `created_at` TEXT (ISO)

Нормализация URL: схема+host+path без `?query` и `#hash` (через `urllib.parse`).

## 2. Агент — stateless поверх БД

Checkpointer (MemorySaver) убирается. На каждый вопрос:
1. Загрузить историю сообщений чата из БД.
2. Собрать вход: `[SystemMessage(build_page_system_message(page)), …история, HumanMessage(question)]`.
3. Вызвать агента (`build_agent`), взять `result["messages"][-1].content`.
4. Записать в БД user-сообщение и assistant-сообщение, обновить `chats.updated_at`.

Системный промпт со страницей подставляется свежим каждый раз (страница актуальна).
Длину истории, отправляемой модели, не ограничиваем (демо; чаты короткие).

## 3. API бэкенда

| Метод | Эндпоинт | Тело / параметры | Ответ |
|---|---|---|---|
| GET | `/chats` | `?url=<page_url>` (опц.) | `{ page: ChatMeta[], all: ChatMeta[] }` — для страницы и все; pinned первыми, затем по `updated_at` desc |
| POST | `/chats` | `{ page_url, page_title }` | `ChatMeta` (новый чат) |
| GET | `/chats/{id}` | — | `{ chat: ChatMeta, messages: Message[] }` |
| POST | `/chats/{id}/messages` | `{ question, page: {title,url,text} }` | `{ answer: str }` |
| PATCH | `/chats/{id}` | `{ pinned?, title?, tags? }` | `ChatMeta` |
| DELETE | `/chats/{id}` | — | `{ ok: true }` |

`ChatMeta` = `{ id, page_url, page_title, title, pinned, tags: string[], created_at, updated_at, preview }`
(`preview` — обрезок последнего сообщения). `Message` = `{ id, role, content, created_at }`.

Старый `POST /chat` удаляется; логика вызова агента переезжает в `POST /chats/{id}/messages`.
CORS и обработка ошибок агента — как раньше (ошибка → текстовый ответ, не 500).
`GET /health` сохраняется.

## 4. Фронтенд

- **Выезжающий сайдбар слева** в боковой панели; кнопка-бургер в шапке открывает/закрывает.
- Сайдбар: **«+ Новый чат»**; секции **«Эта страница»** и **«Все чаты»**; закреплённые (📌) — наверху каждой секции; строка чата = заголовок + превью + теги + действия 📌 / 🗑.
- **Фильтр по тегу**: клик по тегу фильтрует список, повторный клик сбрасывает.
- **Теги активного чата**: в шапке чата — добавление (поле/Enter) и удаление тега.
- При открытии панели: открывается последний чат текущей страницы; если нет — пустой новый чат (создаётся в БД при первом сообщении).
- Модули:
  - `src/lib/chatsApi.ts` — клиент CRUD (типы `ChatMeta`, `Message`).
  - `src/lib/page.ts` — добавить нормализацию URL (или нормализовать на бэкенде; нормализация ключа — на бэкенде, фронт шлёт сырой URL).
  - `src/Sidebar.tsx` — список чатов, секции, фильтр по тегу, действия.
  - `src/ChatPanel.tsx` — активный чат + теги в шапке + бургер (рефакторинг текущего).
  - `src/App.tsx` — корневой стейт (текущий чат, список, страница), оркестрация.
  - `src/main.tsx` — рендерит `App`.

## 5. Обработка ошибок

- Сервер недоступен / ошибка агента → сообщение в ленте чата (как сейчас).
- Ошибки CRUD (список/закреп/тег/удаление) → ненавязчивая строка ошибки в сайдбаре.
- Удаление чата → подтверждение перед `DELETE`.

## 6. Тестирование

- Бэкенд (`pytest`): нормализация URL; CRUD чатов (create/list/get/patch/delete);
  `/chats/{id}/messages` с моком агента (проверка, что история из БД участвует во вводе
  и оба сообщения сохраняются); список — pinned первыми.
- Фронт: сборка (`npm run build`, tsc) + ручная проверка. JS-тест-раннер не вводим (YAGNI).

## Принятые решения

- Хранилище: SQLite на бэкенде (единственный источник правды).
- Привязка к странице: нормализованный URL без query/hash.
- Навигация: выезжающий сайдбар слева (ChatGPT-стиль).
- Агент stateless; история берётся из БД; системный промпт со страницей — свежий каждый запрос.
